BAKOELAPTOP adalah platform marketplace untuk jual beli laptop bekas dengan
sistem approval workflow, role-based access control, dan shopping cart.

Tech Stack:
- Frontend: HTML5, CSS3, JavaScript (Vanilla)
- Backend: Firebase (Authentication, Firestore Database)
- Storage: Base64 encoding untuk gambar produk
- Deployment: Local development server (Live Server)

================================================================================
                    TECHNICAL CHALLENGES & SOLUTIONS
================================================================================

CHALLENGE 1: Navbar Auth State Flicker
Problem: Tombol Sign In/Sign Up muncul sebentar, lalu berubah ke Profile/Cart
         setelah user login (terlihat seperti bug)

Solution: 
- Sembunyikan semua navbar buttons saat load (display: none)
- Tunggu Firebase Auth onAuthStateChanged trigger
- Tampilkan button yang sesuai dengan smooth fade-in transition
- Hasil: Tidak ada flicker, UX lebih smooth

CHALLENGE 2: Composite Index Requirement
Problem: Firebase error "requires a composite index" saat query
         products.where('status', '==', 'approved').orderBy('createdAt')

Solution:
- Buat composite index di Firebase Console
- Index: status (ASC) + createdAt (DESC)
- Tunggu index build selesai (~5 menit)
- Hasil: Query berjalan lancar, loading lebih cepat

CHALLENGE 3: Slow Product Loading
Problem: Products loading lama karena base64 images besar

Solution (Sementara):
- Tampilkan loading message yang informatif
- Gunakan single where clause untuk reduce query time
- Filter sold products di client-side
- Future: Compress images atau gunakan Firebase Storage

CHALLENGE 4: Register Redirect Problem
Problem: Setelah register, user di-redirect tapi navbar masih tampil
         Sign In/Sign Up (padahal seharusnya sudah login)

Solution:
- Pindahkan redirect logic ke firebase-auth.js
- Tambah delay 500ms untuk auth state propagation
- Pastikan Firebase Auth persistence tersimpan sebelum redirect
- Hasil: User langsung terlihat login setelah register

CHALLENGE 5: Error Message Tidak Jelas
Problem: Error messages dari Firebase terlalu teknis dan dalam English

Solution:
- Buat switch-case untuk semua Firebase error codes
- Translate ke Bahasa Indonesia
- Tambah emoji untuk visual feedback (‚ùå, ‚úÖ)
- Styling error box dengan background merah
- Hasil: User lebih mudah memahami kesalahan

RELASI DETAIL:
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ

1. USERS ‚Üí PRODUCTS (1:N)
   - Satu user bisa menjual banyak products
   - Field: products.sellerId = users.uid
   - Type: One-to-Many

2. USERS ‚Üí CARTS (1:1)
   - Satu user punya satu cart
   - Field: carts.userId = users.uid
   - Type: One-to-One

3. PRODUCTS ‚Üí CART ITEMS (1:N - referenced)
   - Satu product bisa ada di banyak cart
   - Field: cartItems.productId = products.id
   - Type: Referenced (denormalized data)
   - Note: Cart menyimpan copy data product (name, price, image)
           untuk performa & konsistensi

4. CARTS ‚Üí CART ITEMS (1:N - embedded)
   - Satu cart berisi banyak items
   - Type: Embedded array di dalam cart document
   - Advantage: Single read operation, faster loading

  - Type: boolean

  - Constraint: NOT NULL

products.price:
  - Type: number

products.stock:
  - Type: number
  - Constraint: >= 0


INDEXES:
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ

1. users.email - Automatic (Auth)
2. users.uid - Primary Key
3. products.id - Primary Key

5. products.sellerId - Single Field Index (untuk query by seller)
6. carts.userId - Primary Key


BUSINESS RULES:
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ

   - Status 'approved' + sold=false: Visible di Buy page
   - Status 'approved' + sold=true: Not visible
   - Status 'rejected': Not visible

   - Cart dibuat otomatis saat user pertama kali add item
   - Quantity max: product.stock
   - Remove item jika quantity = 0
   - Cart tetap ada setelah logout (persistence)

3. Role Authorization:
   - Admin: Access ke admin.html, approve/reject, delete
   - User: Access ke sell.html, cart.html, profile.html
   - Guest: Access ke index.html, buy.html, login, register
